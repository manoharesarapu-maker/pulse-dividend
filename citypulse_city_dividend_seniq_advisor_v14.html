Sales enablement · how to use the City Dividend calculator

Run a value conversation, not a product demo.

This tab gives sales teams the talk track, discovery flow, calculation explanation and safe language needed to use the CityPulse City Dividend calculator with council CEOs, CIOs, Directors, Finance, Infrastructure, Community, Economy/Place and Operations leaders.

Purpose

What the tool is

A configurable value engine that converts council pain points into a transparent City Dividend across Financial, Time, Community, Economic, Climate and Safety outcomes.

Positioning

What the tool is not

It is not a guaranteed ROI promise, a finance-only spreadsheet, or a fine-revenue machine. It is a structured value hypothesis to validate with council data.

Mindset

The golden rule

Let the council own the assumptions. The strongest sales moment is not a high number; it is a trusted number the council recognises.

The simple sales talk track

“Let’s start with the operating pressures you are already feeling — parking, kerbside pressure, asset condition, community sentiment, economic vitality, safety, climate or reporting. We then configure CityPulse around those pressures and make every assumption visible so your leaders can challenge, tune and validate the value case.”

How to run the conversation

Start with pain, not products.Ask where the council is losing time, revenue, trust, safety confidence, economic activity or asset visibility.
Select the council profile.Use it as a starting point only. Explain that profiles seed assumptions; they are not benchmarks carved into stone tablets.
Select pain points.Choose only the pressures that genuinely apply. A smaller, credible value story beats a Christmas tree of every possible benefit.
Review product use cases.Explain how CityPulse, SenFORCE, SenPIC, SenMAP, SenDISA, SenBOS, SenIQ, Community and Economic Intelligence combine.
Tune assumptions openly.Invite the customer to challenge every value: uplift rates, cost per hour, event attendance, spend attribution, maturity and deployment scale.
Interpret the City Dividend.Separate validated financial value from broader council value. Do not collapse trust, safety and economic vitality into “cash savings.”
Finish with validation.Agree the data owners needed for a validation workshop: Finance, Local Laws, Assets, Community, Economy/Place, Operations and ICT.

How the maths works — plain English

The calculator uses a transparent pattern: addressable value × impact assumption × deployment scale × scenario × maturity × ramp. Costs are modelled separately by product, including one-off and recurring costs.

Language that keeps us credible

Say this

  • “This is an indicative value case, not a guaranteed saving.”
  • “The council should validate assumptions with local data owners.”
  • “Some value is cashable; some is capacity, trust, safety or economic value.”
  • “The strongest benefit comes when signals are fused across operations, community and economy.”

Avoid this

  • “This proves the ROI.”
  • “CityPulse will pay for itself in every council.”
  • “This is just about parking revenue.”
  • “All time released becomes a budget saving.”
  • “SenPIC is mainly school zones.”

Admin guide · configuration-first model

Change assumptions without rebuilding the calculator.

The calculator is designed around a JSON-backed configuration. Sales and solution leaders can tune assumptions, formulas, costs, scenarios, maturity settings and language, then export the configuration for reuse.

What admins can change directly

  • App title, subtitle, discount rate and horizon.
  • Scenario names and multipliers.
  • Maturity names and multipliers.
  • Parameter values and cost inputs.
  • Benefit formulas, cashability, confidence and ramp profiles.
  • Full JSON configuration export/import.

What is best changed in JSON

  • Products and product descriptions.
  • Use cases and product mapping.
  • Pain point wording and recommended use cases.
  • Dividends and benefit weightings.
  • Ramp profiles and overlap notes.
  • Output language and validation text.

Recommended GitHub Pages setup

  1. Download this HTML file and rename it index.html.
  2. Create a GitHub repository, for example citypulse-city-dividend.
  3. Upload index.html.
  4. Go to repository settings → Pages → deploy from main branch.
  5. Share the GitHub Pages URL with sales teams.
Tip: Keep customer-specific scenarios out of the public repo unless the assumptions are fictional or approved for sharing.

Admin caution

This is a single-file Phase 1 site. The admin panel is useful for configuration, but it is not secure. Do not treat it as production access control. For Phase 2, move configuration, authentication, saved scenarios and PDF generation to a backend.
